# iPhone自動操作プロジェクト (iphone-auto) 仕様・計画書

## 1. プロジェクト概要
macOSの「iPhoneミラーリング」機能を介して、iPhoneアプリ（特に「プラリー」）をAIエージェントが自動操作するための基盤開発。

## 2. 現在のステータス
- **フェーズ1: コアフレームワークの実装完了**
  - `uv` によるプロジェクト管理のセットアップ。
  - `window_helper.py`: AppleScriptによる「iPhoneミラーリング」ウィンドウの自動座標取得。
  - `vision.py`: ウィンドウ内限定のキャプチャと画像認識（OpenCV）の基盤。
  - `control.py`: ウィンドウ相対座標でのタップ・スワイプ操作（PyAutoGUI）。
  - `agent.py`: 状態遷移（State Machine）に基づいたメインループの枠組み。
- **デバッグ・運用性の向上 (New)**
  - **リアルタイム表示**: 実行中、ターミナルにヒットした画像名と信頼度を表示する機能を追加。
  - **実行エントリーポイント**: ルートディレクトリの `main.py` からアプリ名を指定して起動可能に統合。
  - **可視化ログ**: マッチング箇所を赤枠で囲んだ画像を、原因となったテンプレート画像名を含めて `logs/` に自動保存。

## 3. システムアーキテクチャ
- **エントリーポイント**: `main.py` (引数でアプリ名を切替可能)
- **認識**: `screencapture` (macOS) + `OpenCV` (テンプレートマッチング)
- **制御**: `PyAutoGUI` (マウス操作のシミュレーション)
- **座標管理**: AppleScript により、ウィンドウが移動しても追従可能

## 4. 実行方法
```bash
# プラリー(pully)の自動化を開始
python main.py

# トリマ(torima)の自動化を開始
python main.py torima
```

## 4. 今後の計画 (Next Steps)
1. **テンプレート画像の収集**:
   - アプリ内の主要な状態（ホーム、広告表示中、エラー等）を示すアイコンやボタンの画像を `images/` に保存。
2. **状態判定ロジックの実装**:
   - `agent.py` の `update_state` メソッドを更新し、収集した画像を用いて現在の画面状態を特定。
3. **操作ロジックの定義**:
   - `agent.py` の `decide_action` に、各状態に対する具体的な操作（タップ位置、待機時間）を記述。
4. **フェイルセーフの実装**:
   - 予期せぬダイアログが出た場合や、一定時間状態が変わらない場合の例外処理。

## 5. 注意事項
- **実行権限**: `screencapture` や `pyautogui` の実行には、macOSの「アクセシビリティ」および「画面収録」の許可が必要です。
- **ウィンドウ名**: 「iPhone Mirroring」という名前のウィンドウが存在することを前提としています。
